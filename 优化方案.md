记录日期：2026-07-01

## 一、架构重构

### 1. 模块化拆分
当前 1477 行全在 `main.py`，拆分为：

| 模块 | 职责 |
|------|------|
| `i18n.py` | LANG 字典 + t() / set_language() |
| `network.py` | 跨平台网络检测 |
| `speedtest.py` | 下载测速核心逻辑 |
| `settings.py` | SettingsDialog 类 |
| `ui.py` | MetricCard、SpeedTester UI |
| `config.py` | load_config / save_config |
| `main.py` | 入口（约 50 行） |

### 2. 国际化数据分离
将 LANG 字典移到独立 JSON 文件：

```
i18n/
  en.json
  zh.json
  fr.json
  de.json
  es.json
  pt.json
  it.json
  ru.json
  pl.json
  ar.json
```

按需加载，减少主代码体积。

## 二、测速算法

### 3. 多连接并行下载
- 当前：单连接 HTTP GET + iter_content（CHUNK_SIZE=65536）
- 问题：>500 Mbps 时单连接无法打满带宽
- 方案：4-8 个 Range 请求分片下载，汇总速度
- 前提：服务器需要支持 Range 请求

### 4. 渐进式文件选择
- 10MB → 50MB → 200MB 自动切换
- 低速连接在 10MB 后结束，不浪费带宽
- 高速连接自动升级到更大文件保证准确性

### 5. 上传测速
- 通过 POST multipart 上传随机数据
- 需要有配套的测试服务器

## 三、IP 信息获取

### 6. 并发请求（P0）
- 当前：串行 fallback（ip-api.com → ipinfo.io → ipify.org）
- 方案：concurrent.futures 同时请求 3 个服务，取最快响应
- 价值：平均 0.5s 显示 IP vs 当前可能 10s+

### 7. ip-api.com HTTPS
- 当前使用 HTTP（免费限制）
- 可考虑改用 ipinfo.io 为主服务（HTTPS）

## 四、网络检测

### 8. 替代 PowerShell（P2）
- 当前：启动 64-bit PowerShell 进程执行 PS 脚本（冷启动 1-2s）
- 方案 A：pywin32 WMI 直接查询（推荐，pyinstaller 已打包 pywin32）
- 方案 B：Windows SDK GetAdaptersAddresses via ctypes

### 9. 测速期间暂停网络刷新（P0）
- 当前：每 10s 刷新一次网络信息，测速时也刷新
- 方案：检测 `self.downloading`，测速期间跳过刷新

## 六、打包/构建

### 14. Nuitka 替代 PyInstaller（P3）
- PyInstaller：启动 1-3s，体积 20-36MB，反编译容易
- Nuitka：启动 0.2s，体积 15-28MB，反编译难
- 需验证 tkinter + requests 兼容性

### 15. 自动版本号（P2）
- 从 git tag 自动注入 VERSION
- build.py 中 `git describe --tags --always`
